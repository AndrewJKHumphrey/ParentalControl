ParentGuard Help

Everything you need to set up and manage parental controls on Windows.

Quick Start

ParentGuard runs as a background Windows service that enforces your rules 24/7, plus an admin UI that only you can access.

Run ParentGuardSetup.msi and follow the installer wizard. Administrator rights are required.

Open ParentalControl.UI.exe and log in with the default password parent1234.

Go to Profiles and add a profile for each child, entering their Windows username.

Configure Screen Time, App Control, and Web Filter limits per profile.

Rules take effect immediately — no reboot required. Check the Dashboard to monitor activity.

Tip: Change the default password immediately in Settings → Change Password before a child gets access to the machine.

Login & Password

The UI is protected by a password so children cannot change their own rules.

  • Default password: parent1234
  • Change it in Settings → Change Password (minimum 6 characters).
  • Wrong password is rejected with an error — the UI does not open.
  • If you forget your password, delete %ProgramData%\ParentalControl\data.db and re-run ParentGuardSetup.msi (this resets all data).

Dashboard

The Dashboard shows a real-time overview of the active profile and lets you make quick adjustments.

Status Cards

  • Screen Time Remaining — time left today for the active user. Colour-coded: green >30 min, yellow <30 min, red <5 min, purple = locked.
  • App Time — time left for Screen-Time-Only apps today.
  • Focus Mode — whether a focus window is currently active.
  • Web Filter — number of sites blocked today.

Enforcement Toggles

Each feature can be individually enabled or disabled. Admin Mode disables all four at once — useful when you want to temporarily lift all restrictions.

Time Adjustments

Allowed Range +60Extends the allowed hours window end time by 60 minutes for today.
App Time +30Adds 30 bonus minutes for Screen-Time-Only apps today.

Profiles

Each profile maps to a Windows user account. Rules (screen time, app control, web filter) are enforced based on which user is logged in.

Default Profile

A built-in Default profile (blank Windows username) acts as a catch-all. Any logged-in user with no specific profile uses its rules.

Adding a Profile

Click Add Profile and enter a display name (e.g. "Lily") and the child's exact Windows username.

ParentGuard automatically creates 7-day screen time and focus schedules for the new profile.

Set rules on the Screen Time, App Control, Focus Mode, and Web Filter pages for this profile.

Profile Options

  • IsEnabled — toggle off to temporarily suspend all enforcement for that profile.
  • AlwaysRelock — when on, the session is locked on every new login once the daily limit is reached, not just the first time.

Screen Time

Limits how long a child can use the computer each day, and restricts usage to certain hours.

Daily Limit

Enter minutes per day. Set to 0 for unlimited. When the limit is reached the session is locked.

Allowed Hours

Set Allowed From and Allowed Until to define when the computer can be used. Outside these hours the session is locked.

Accepted time formats: 0800, 8:00, 8am, 8:30pm, 20:30.

Tip: Use Save to All to apply the same schedule across every profile at once.
Note: Spans-midnight ranges (e.g. 10 pm – 6 am) are not currently supported. Set Allowed From earlier than Allowed Until.

App Control

Controls which applications a child can run and for how long.

Access Modes

Block  App is killed within 5 seconds of launch. Logged in Activity Log.

Screen Time Only  App runs but counts toward the daily screen-time limit. Killed when the limit is reached or outside allowed hours.

Allow  App is unrestricted.

App Time Limits

Separate per-day limits can be set for Screen-Time-Only apps. Set to 0 for unlimited.

Scanning for Apps

  • Quick Scan — reads Steam's game library to find installed games instantly.
  • Deep Scan — recursively scans drives for executables. Takes longer but finds everything.
  • Apps rated by RAWG (requires an API key in Settings) will show ESRB rating, genres, and tags automatically.
  • Scanning auto-configures rules based on your ESRB and genre settings. Any entry you manually change is flagged Modified and won't be overwritten on future scans.

Focus Mode

During focus windows, only apps explicitly marked Allowed in Focus Mode can run. Everything else is blocked — ideal for homework or bedtime.

Setting a Focus Schedule

Set Focus From and Focus Until for each day. Enable Focus Mode on the profile's toggle to activate enforcement.

Mark individual apps as allowed during focus from the App Control page (the In Focus column).

Tip: Browsers and games are typically not in focus; educational apps and Office tools typically are.

Browser Extension

The ParentGuard Web Filter extension enforces web filtering rules in Edge and Chrome, showing a block page whenever a site is blocked. It must be loaded manually as an unpacked extension using Developer Mode.

Installing in Microsoft Edge

Open Edge and navigate to edge://extensions/.

Enable Developer mode using the toggle in the bottom-left corner.

Click Load unpacked and select the ParentalControl.Extension folder (located in the ParentGuard installation directory, e.g. C:\Program Files\ParentGuard\extension).

The ParentGuard Web Filter extension will appear in the list. Confirm it is enabled.

Installing in Google Chrome

Open Chrome and navigate to chrome://extensions/.

Enable Developer mode using the toggle in the top-right corner.

Click Load unpacked and select the ParentalControl.Extension folder.

The ParentGuard Web Filter extension will appear in the list. Confirm it is enabled.

Important: Developer mode must remain enabled in the browser for the extension to stay active. If a child disables Developer mode, the extension will be removed and web filtering will stop working.
Tip: After loading the extension, close and reopen the browser to ensure it is fully active.

Web Filter

Blocks websites in Edge and Chrome using the ParentGuard browser extension.

Manual Domain Rules

  • Paste a full URL or domain — it's automatically stripped to just the hostname (e.g. youtube.com).
  • Set the rule to Block or Allow.
  • Sub-domains are matched automatically: blocking example.com also blocks sub.example.com.

Allow Mode

When enabled, only domains on the Allow list are accessible. Everything else is blocked. Useful for young children who should only visit specific sites.

Content Tags

Pre-built domain lists for categories like Adult Content and Gambling. Enable a tag for a profile to block thousands of domains automatically.

  • Sync All — downloads the latest domain lists from their source URLs.
  • Copy Tags to All Profiles — applies the same tag selections to every profile.

How Blocking Works

Web filter rules are enforced by the ParentGuard browser extension. Rules sync within 1–2 seconds of saving in the UI, and a block page is shown in place of a browser error when a site is blocked.

Activity Log

A searchable history of enforcement events.

Event TypeWhat it means
ServiceStartedThe enforcement service started.
ScreenTimeLockedA user's session was locked due to a time limit.
AppBlockedA blocked application was terminated.
WebsiteBlockedA web request was blocked by the extension.

Use the Date, Type, and Search filters to narrow down entries. Clear Log permanently deletes all entries after confirmation.

Settings

Appearance

  • Theme — choose from Default, Deep Ocean, and more. Debug builds also include Raven and Custom.
  • Dark / Light Mode — switches the colour palette within the selected theme.
  • UI Scale — adjust for 1080p, 1440p, or 4K displays.
  • Time Format — 12-hour (8:00 PM) or 24-hour (20:00) display throughout the app.

Notifications

Get notified when the session is locked or an app is blocked. Three delivery modes:

  • Email — SMTP credentials required.
  • SMS — phone number + carrier email gateway (e.g. 5551234567@txt.att.net).
  • Ntfy — free push notifications via ntfy.sh. Enter your topic name.

RAWG API Key

Used to fetch ESRB ratings, genres, and tags for apps during scanning. Get a free key at rawg.io and paste it here.

Troubleshooting

Service shows "Stopped" in the sidebar

Click Start Service in the sidebar to start it manually.

If it stops again immediately, re-run ParentGuardSetup.msi. This re-registers the service and adds Windows Defender exclusions needed for it to start.

Check the Windows Event Viewer → Application log for errors from source ParentalControl.

Browser extension not appearing in Edge or Chrome

Open the browser's extensions page (edge://extensions/ or chrome://extensions/) and confirm Developer mode is still enabled — it must remain on for the unpacked extension to stay loaded.

If the extension is missing, click Load unpacked and re-select the ParentalControl.Extension folder. See the Browser Extension section for full steps.

Close all browser windows completely (including background processes in Task Manager) and relaunch the browser.

Web filter not blocking sites

  • Confirm the browser extension is installed and enabled (edge://extensions/).
  • Check that the Web Filter toggle is enabled on the Dashboard for the active profile.
  • Verify the domain rule is saved: go to Web Filter, select the correct profile, and confirm the rule appears.
  • For Content Tags, click Sync All to ensure the latest domain list is downloaded.

App is not being blocked

  • Confirm App Control is enabled on the Dashboard.
  • Check the app's Access Mode is set to Block for the correct profile.
  • The service only monitors processes launched after it starts — restart the app after enabling the rule.
  • Allow up to 5 seconds after launch for the block to take effect.

Screen time not locking

  • Ensure Screen Time is enabled on the Dashboard.
  • Confirm a Daily Limit greater than 0 is set for the active profile on the Screen Time page.
  • Verify the Windows username on the profile matches the logged-in user exactly (case-insensitive).

Forgot admin password

Delete C:\ProgramData\ParentalControl\data.db and re-run ParentGuardSetup.msi. This wipes all settings and restores the default password parent1234.

Warning: Deleting the database removes all profiles, rules, and activity history.
